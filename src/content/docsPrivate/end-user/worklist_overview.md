The **Worklist** gives users a consolidated view of all approvals assigned to them across all supported transaction types.  
This guide explains how to access the Worklist, how to act on items, and how to use filtering and sorting options.

## 1. Overview

The Worklist allows approvers and managers to:
- View all pending approvals assigned to them  
- Sort and filter by transaction type, aging, or amount  
- Open transactions directly in NetSuite  
- Track SLA deadlines  
- Monitor escalations and exceptions  

The Worklist is available to all users with approval responsibilities.

## 2. Accessing the Worklist

Users can access the Worklist from:

**Navigation**
- **Approvals → My Worklist**

**Behavior**
- Displays all active approval steps assigned to the user  
- Updates automatically when queue processing completes  
- Allows quick access to the Approval Drawer within each transaction  

## 3. Worklist Layout

The Worklist includes sortable columns to help users identify priority items.

### 3.1 Standard Columns

**Fields**
- Transaction Number  
- Vendor / Customer  
- Amount  
- Transaction Date  
- Assigned Step  
- Age / SLA Proximity  
- Exceptions (if applicable)  

### 3.2 Behavior

**Behavior**
- Users can sort columns (e.g., by amount or aging).  
- Exception icons appear when an item requires special attention.  
- Clicking a row opens the transaction with the Approval Drawer active.  

## 4. Filtering Worklist Items

The Worklist includes filtering options to narrow down results.

### 4.1 Filter Types

**Fields**
- Date range  
- Transaction type  
- Subsidiary  
- Department  
- Exception presence  

### 4.2 Behavior

**Behavior**
- Filters apply instantly.  
- Multiple filters can be combined.  
- Clearing filters restores the full list.  

## 5. Taking Action from the Worklist

Users can approve or reject items by opening the associated transaction.

### 5.1 Opening a Transaction

**Behavior**
- Clicking a Worklist item opens the record in a new tab.  
- The Approval Drawer loads automatically.  

### 5.2 Acting on an Item

**Behavior**
- Users approve or reject from the Drawer.  
- Timeline updates immediately.  
- Worklist refreshes during the next queue cycle.  

## 6. Worklist Indicators

The Worklist helps identify high-priority or at-risk approvals.

### 6.1 SLA Indicators

**Fields**
- Approaching deadline  
- Late approval  
- Critical lateness  

### 6.2 Exception Indicators

**Fields**
- SoD warnings  
- Assignment issues  
- Budget warnings  
- Data errors  

### 6.3 Behavior

**Behavior**
- Indicators are visual only; action happens inside the Drawer.  
- Items with severe issues may appear red or flagged depending on theme.  

## 7. Tips for Managing Workload

**Tips**
- Sort by “Age” to handle oldest approvals first.  
- Use filters to focus on specific subsidiaries or departments.  
- Address exception-flagged items early.  
- Use comments to provide clarity or rationale to submitters or admins.  

## 8. Troubleshooting Worklist Issues

### Common Issues

**Items Not Updating**
- Queue processing delay  
- Saved search caching  

**Missing Worklist Items**
- User may not be the assigned approver  
- Delegation may be active  
- Record permissions may be insufficient  

**Incorrect SLA Indicators**
- Report Settings thresholds outdated  
- SLA calculation using working-hours mode
